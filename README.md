# Pavel Vault

Flutter-клиент для существующего Obsidian vault. Приложение не вводит собственный публичный формат: серверной копией остаются обычные Markdown/YAML-файлы и вложения на WebDAV.

## Что реализовано

- Android, Linux и Windows runners.
- Первый вход с проверкой WebDAV и сохранением секрета в системном secure storage.
- AES-256-GCM шифрование локальной копии, блокировка после настраиваемой задержки при уходе в фон, биометрия/Windows Hello и PIN fallback.
- Двусторонняя синхронизация по ETag с общей базовой версией файла и ручным экраном конфликтов.
- Точечная автоматическая отправка в WebDAV после редактирования или создания заметки без полного обхода хранилища; ETag защищает от перезаписи конфликтующей версии, а при недоступной сети локальная зашифрованная копия сохраняется.
- Полная и точечная синхронизации выполняются через общую очередь; повторные изменения одного файла объединяются, а переключение WebDAV-профиля дожидается завершения старой очереди.
- Глобальный экранный статус сохранения и WebDAV-отправки, видимый поверх редактора; полная ручная синхронизация показывает этап, счётчик и процент.
- Адаптивная навигация для desktop, телефонов и узкого внешнего экрана Galaxy Fold.
- Нативные коллекции и формы добавления книг, рецептов, растений, чая, аптечки и обычных заметок.
- Нативное создание проектов и всех проектных материалов из Obsidian: задач, общих заметок, встреч, решений, брифов, справочников и отчётов; отдельная вкладка показывает материалы всех типов.
- Главная страница повторяет структуру `Onboarding.md`/`home-dashboard`: KPI, фокус-задачи, быстрый доступ, проекты в движении, здоровье за 7 дней, последние отчёты и заметки.
- Анимированные переходы между экранами, появление секций главной, смена полей проектных форм и график тренировок.
- Поля тренировки меняются для велосипеда, гребли, скакалки и тенниса в соответствии с Obsidian-шаблоном.
- Фильтрация проектов по активным/архивным и сортировка по статусу, имени или изменению.
- Поиск и просмотр всего vault, гибкая сортировка коллекций по имени, датам, рейтингу и типу, Obsidian wiki-ссылки, attachments, Markdown preview и запасной source editor.
- Точечное изменение YAML и Markdown-секций без перезаписи неизвестного содержимого.
- Формы выполненной работы и тренировок, проекты и задачи из существующего frontmatter.
- Переключение даты во вкладке «День» и добавление данных в выбранный день.
- Быстрый календарь ежедневника на главном экране: существующие даты отмечены точками и открываются без дублирования, пустые даты ведут в нативную форму создания на основе Obsidian-шаблона.
- Недельные, месячные и годовые отчёты повторяют DataviewJS-логику Obsidian: KPI, ежедневники, привычки, выполненное, тренировки, задачи, новые заметки, качество данных, таблицы и интерактивные графики; доступны Markdown snapshot, PDF и CSV.
- PDF/CSV/Markdown-экспорт включает шаги, сон, дневные калории, задачи и полный набор тренировочных метрик (`Aerobic TE`, `Anaerobic TE`, TRIMP, нагрузку, восстановление и специфичные для спорта поля); PDF использует кириллический Unicode-шрифт.
- В тренировочной заметке можно одним нажатием скопировать полный исходный YAML frontmatter.
- Страницу отчётов можно собирать из готовых и пользовательских блоков: менять порядок и видимость, задавать источник, фильтры, группировку, построчные формулы, агрегацию и тип графика. Раскладка синхронизируется через `.pavel-vault/reports-layout.v1.json`.
- Настройки поддерживают несколько WebDAV-профилей с отдельными зашифрованными кешами, безопасное переключение после синхронизации и миграцию существующего подключения.
- Внешние изображения и вложения vault доступны через единый офлайн-кеш; в управлении памятью показывается размер по категориям, доступны лимит LRU и очистка.
- Выразительные переходы, анимации карточек, графиков и загрузки можно переключить на умеренные или минимальные; системный Reduce Motion имеет приоритет.
- Нативная заглушка для известных DataviewJS-блоков без исполнения произвольного JavaScript.

## Запуск

Требуется Flutter 3.44+ и Dart 3.12+.

```bash
flutter pub get
flutter run -d linux
# или: flutter run -d windows
# или: flutter run -d <android-device>
```

Для Linux необходимы GTK development packages и `libsecret` с запущенным Secret Service (`gnome-keyring`, KWallet или аналог). Для Android используется minimum SDK 24.

## Проверка

```bash
flutter analyze
flutter test
dart run tool/vault_audit.dart ..
flutter build linux --debug
```

GitHub Actions выполняет `flutter pub get`, `flutter analyze` и `flutter test` на Linux. Файловые тесты атомарного encrypted store дополнительно запускаются на Windows.

При `flutter run` debug-консоль показывает этапы подключения, HTTP-методы и коды WebDAV, загрузки файлов, решения синхронизации и ошибки со stack trace. Пароли, PIN и `Authorization` не логируются.

Аудит читает реальные заметки без записи и проверяет, что каждая Markdown-заметка разбирается моделью приложения.

## Формат совместимости

- Daily notes: `Daily/DD MMMM YYYY.md`, поля `created`, `week`, `step`, `sleep`, `calories`.
- Учёт работы: `- Описание - 1.5ч - #Project` внутри секции `Что было сделано`.
- Тренировки: `Areas/Health/Traning/YYYY-MM-DD/{sport_key}.md`, `type: training-log`, `metrics` и `assessment`.
- Проекты: `Projects/<project>/_Project.md`; задачи и заметки используют текущие `type`, `project`, `status` и `note_type`.
- Отчёты: `Resources/Reports/{Weekly|Month|Year}`, `type: period-report`.

Приложение не исполняет сторонние плагины Obsidian и произвольный DataviewJS. Исходный код таких блоков сохраняется и продолжает работать в Obsidian.

## Безопасность синхронизации

Удалённый WebDAV считается незашифрованным, как выбрано для этого vault. Публичные подключения разрешены только по HTTPS. HTTP можно использовать для `localhost`, private/link-local IP и имён `.local`; интерфейс явно предупреждает, что трафик в этом случае передаётся без TLS.

PIN защищает доступ к интерфейсу и служит fallback для разблокировки. Он не является ключом шифрования локального кеша. AES-256-GCM master key создаётся отдельно и хранится через системное secure storage.

Перед первой реальной синхронизацией рекомендуется сделать серверный snapshot. При конкурентном изменении приложение не выбирает победителя автоматически: обе версии остаются доступны до ручного решения.

## Восстановление локального кеша

Зашифрованные `.pvo`-объекты записываются через временный файл и атомарный `rename`. Manifest имеет версию формата, checksum и резервную копию. Batch-операции используют journal: после аварийного завершения кеш восстанавливается в согласованное состояние через roll-forward.

При нормальном запуске приложение доверяет проверенному manifest и не запускает дополнительную полную integrity-сверку. Полное сканирование выполняется после повреждения manifest, восстановления из backup, обнаружения незавершённого batch или вручную через «Настройки → Проверить целостность кеша».

Известные ограничения:

- восстановление гарантирует согласованность manifest и доступных объектов, но не возвращает данные, если одновременно повреждены сам объект и все его внешние копии;
- атомарность `rename` зависит от гарантий файловой системы, поэтому соответствующие тесты выполняются отдельно на Linux и Windows;
- HTTP даже в локальной сети не защищает содержимое WebDAV от перехвата;
- сторонние плагины Obsidian и произвольный DataviewJS в приложении не исполняются.
